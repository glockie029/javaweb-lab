# JavaWeb 审计流程与案例复盘模板

该文档用于把零散知识转化为一套可执行的审计方法。目标是让每次审计都沿同一条主线推进，并能留下可复查的记录。

## 1. 标准审计流程

### 第一步：识别工程与运行形态

1. 看 `pom.xml` 或 `build.gradle`
2. 识别技术栈、模块结构、打包方式、profile
3. 枚举核心依赖和高风险组件

输出要求：

1. 技术栈摘要
2. 模块关系图
3. 高风险依赖清单

### 第二步：定位入口点

1. 找 `Servlet`、`Filter`、`Listener`
2. 找 `Controller`、`Interceptor`、安全过滤链
3. 找任务入口、消息消费入口、管理入口

输出要求：

1. 入口类型列表
2. 关键路径和未受保护路径

### 第三步：跟踪参数流

1. 标记所有用户可控输入
2. 跟踪输入进入 DTO、Service、DAO、模板、文件、外部请求
3. 同时记录权限检查和校验逻辑

输出要求：

1. source 列表
2. 关键调用链
3. 缺失的校验与权限点

### 第四步：定位 sink 与高风险操作

1. SQL 执行
2. 文件读写
3. 命令执行
4. 反序列化
5. XML 解析
6. 表达式与模板渲染
7. 外部请求

输出要求：

1. sink 列表
2. 与 source 对应的危险链路

### 第五步：复核权限与配置

1. 复核是否只做登录校验而未做资源授权
2. 复核 profile、配置项和管理端点
3. 复核上传目录、日志、调试功能和外部配置

输出要求：

1. 权限链说明
2. 配置风险点列表

### 第六步：形成问题结论

1. 记录问题位置和调用链
2. 判断触发条件和影响范围
3. 评估稳定性、权限要求和利用前提
4. 给出修复建议和回归验证点

## 2. 推荐搜索思路

建议先搜以下内容：

1. 控制器与入口：`@Controller`、`@RestController`、`@RequestMapping`
2. 权限相关：`@PreAuthorize`、`hasRole`、`Filter`、`Interceptor`
3. SQL 相关：`select`、`${`、`createQuery`、`nativeQuery`
4. 文件相关：`MultipartFile`、`File`、`Path`、`Files`
5. 命令与反射：`exec`、`ProcessBuilder`、`Class.forName`、`invoke`
6. 反序列化：`ObjectInputStream`、`readObject`
7. 外连与解析：`RestTemplate`、`WebClient`、`HttpClient`、XML 解析器
8. 模板与表达式：`TemplateEngine`、`SpEL`、`OGNL`、`${`

## 3. 问题记录模板

每个问题建议固定按以下结构记录：

1. 问题名称
2. 漏洞类型
3. 代码位置
4. 入口 source
5. 关键调用链
6. sink
7. 权限要求
8. 触发条件
9. 影响范围
10. 风险等级
11. 修复建议
12. 回归验证点

## 5. 复盘的目标

1. 让同类漏洞在后续项目中可以更快识别。
2. 让审计结论可复查、可交付、可训练。
3. 把零散经验沉淀成稳定的方法论。
