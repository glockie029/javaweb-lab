# Spring 生态与主流框架审计

大多数真实 JavaWeb 项目并不是纯 Servlet/JSP，而是建立在 Spring 生态及其常见持久层、安全框架和模板引擎之上。该文档用于补齐现代 JavaWeb 审计主线。

边界说明：

1. 传统 JavaWeb 与通用 Web 认证、授权、会话安全逻辑，优先放在阶段六学习。
2. 本文档只负责 Spring 生态中的实现差异与框架特有风险，例如 `Interceptor`、`Spring Security`、`Shiro`、前后端分离 `Token` 鉴权实现等。

## 1. Spring Boot

学习重点：

1. 启动类与自动配置机制。
2. 组件扫描、配置注入、Bean 生命周期。
3. 外部配置、环境切换、条件装配。

审计关注点：

1. 自动配置是否引入额外暴露面。
2. 是否开放调试端点、管理端点、监控端点。
3. 配置覆盖是否造成安全策略失效。

## 2. Spring MVC

学习重点：

1. `Controller`、`RequestMapping`、参数绑定、返回值处理。
2. 消息转换器、异常处理、视图解析。
3. `Interceptor`、`Filter` 与 Controller 链路关系。

审计关注点：

1. 入口接口在哪里，参数如何进入对象。
2. 绑定器、转换器、默认值、对象展开是否引入额外输入面。
3. 是否存在白名单路径、未拦截接口或错误的拦截顺序。

## 3. Validation 与对象绑定

学习重点：

1. 参数校验注解与自定义校验器。
2. Bean 绑定、嵌套对象绑定、集合绑定。

审计关注点：

1. 校验是否只覆盖表层字段。
2. 是否存在未校验字段、默认值绕过、对象注入或批量赋值问题。

## 4. Jackson、Fastjson 与消息转换

学习重点：

1. JSON 到对象的映射过程。
2. 多态类型、日期处理、忽略属性、反序列化特性。
3. 常见 JSON 组件的差异，包括 `Jackson` 与 `Fastjson`。

审计关注点：

1. 是否启用危险类型信息。
2. 是否存在宽松反序列化、未知字段忽略带来的安全边界问题。
3. 是否存在把任意 JSON 直接转为通用对象的设计。
4. 是否使用存在历史高风险特性的 JSON 组件或不安全配置。

## 5. MyBatis

学习重点：

1. Mapper、XML SQL、注解 SQL、动态 SQL。
2. 参数绑定、结果映射、分页和排序。

审计关注点：

1. `${}` 与 `#{}` 的区别是否被正确使用。
2. 动态排序、动态表名、动态列名是否受控。
3. 是否存在批量操作、条件缺失和数据权限缺失。

## 6. JPA / Hibernate

学习重点：

1. 实体、仓库、查询方法、JPQL、原生 SQL。
2. 延迟加载、关联关系、事务边界。

审计关注点：

1. 是否存在原生 SQL 拼接。
2. 是否存在查询条件缺失导致的数据越权。
3. 异常或日志是否暴露内部实体结构。

## 7. 模板引擎

常见对象：

1. `Thymeleaf`
2. `Freemarker`
3. `Velocity`
4. 老项目中的 JSP/EL

审计关注点：

1. 模板变量是否来自用户可控输入。
2. 是否存在模板片段、模板路径、表达式内容可控。
3. 输出是否区分 HTML、JavaScript、URL 等上下文。

## 8. Spring Security

学习重点：

1. 认证、授权、过滤器链、会话管理。
2. 路径匹配、角色控制、方法级权限控制。

审计关注点：

1. 过滤器链是否有忽略路径或顺序问题。
2. 是否存在接口保护缺失、仅前端控制、方法级权限遗漏。
3. 是否关闭了关键安全机制或错误配置了会话策略。

## 9. Shiro

学习重点：

1. Realm、Filter、Subject、会话与 rememberMe。
2. 路径控制和注解权限控制。

审计关注点：

1. rememberMe、反序列化、权限绕过历史问题。
2. Filter 链配置是否遗漏敏感接口。
3. 是否存在认证成功但资源授权缺失。

## 10. Actuator、任务、异步与实时通信

学习重点：

1. 管理端点、健康检查、指标、环境信息。
2. 定时任务、异步任务、消息监听器。
3. `WebSocket`、`SSE` 等实时通信入口。

审计关注点：

1. 是否存在未授权管理端点暴露。
2. 异步执行是否绕过上下文和鉴权。
3. 定时任务和消息消费逻辑是否读取外部可控数据。
4. 实时通信入口是否缺少鉴权、消息校验和订阅边界控制。

## 11. 审计时的阅读顺序

建议顺序如下：

1. 看启动类和配置类。
2. 看安全配置、拦截器、过滤器和异常处理。
3. 看 Controller 入口。
4. 跟到 Service 和 DAO。
5. 复核消息转换器、模板引擎、外部请求和配置项。

## 12. 阶段目标

本阶段完成后，应能做到：

1. 快速识别 Spring 项目中的请求链、权限链和配置链。
2. 从框架组件切入，而不是只盯业务方法。
3. 把常见漏洞与具体 Spring 组件对应起来。
