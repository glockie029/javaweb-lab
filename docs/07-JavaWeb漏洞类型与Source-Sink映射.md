# JavaWeb 漏洞类型与 Source-Sink 映射

该文档用于把“漏洞名称”转化为“输入点、校验点、危险点、框架组件和审计路径”。代码审计时，真正要找的是一条可解释的调用链，而不是只记漏洞分类。

## 1. 基本分析框架

建议每个问题都按以下结构理解：

1. `source`：用户输入或外部可控数据从哪里进入。
2. `validation / sanitization`：经过了哪些校验、过滤、编码或权限检查。
3. `sink`：最终进入了哪个危险操作。
4. `context`：运行环境、框架组件、配置项和权限前提是什么。

## 2. 高优先级漏洞映射

### 2.1 SQL 注入

常见 source：

1. 查询参数
2. JSON 字段
3. 排序字段
4. 表名、列名、条件片段

常见 sink：

1. JDBC 字符串拼接 SQL
2. MyBatis `${}`
3. JPA/JPQL 原生拼接

常见切入点：

1. DAO、Mapper、Repository
2. 动态查询构造器

### 2.2 XSS

常见 source：

1. 表单输入
2. 富文本内容
3. 用户资料字段

常见 sink：

1. JSP/模板直接输出
2. 富文本回显
3. 前端模板拼接

常见切入点：

1. 模板变量
2. 详情页、评论区、管理后台预览

### 2.3 文件上传与路径遍历

常见 source：

1. 上传文件名
2. 下载文件路径
3. 目录参数
4. 解压包条目

常见 sink：

1. 文件保存路径拼接
2. 文件读取、删除、预览
3. 压缩包解压

常见切入点：

1. 上传接口
2. 导出下载接口
3. 资源预览接口

### 2.4 命令执行

常见 source：

1. 系统命令参数
2. 脚本内容
3. 路径、文件名、主机名等拼接参数

常见 sink：

1. `Runtime.exec`
2. `ProcessBuilder`
3. 脚本引擎执行

常见切入点：

1. 运维接口
2. 文件处理工具类
3. 第三方系统集成逻辑

### 2.5 反序列化

常见 source：

1. 请求体
2. 缓存数据
3. MQ 消息
4. Session 或 rememberMe 数据

常见 sink：

1. `ObjectInputStream`
2. 危险 JSON 反序列化
3. 第三方组件对象恢复流程

常见切入点：

1. 自定义协议
2. 缓存工具类
3. 安全框架历史兼容逻辑

### 2.6 XXE

常见 source：

1. XML 请求体
2. 配置导入
3. Office/XML 文件解析

常见 sink：

1. 未禁用外部实体的 XML 解析器

常见切入点：

1. 上传导入功能
2. WebService/SOAP 接口

### 2.7 SSRF

常见 source：

1. 可控 URL
2. 回调地址
3. 远程图片、远程文件地址

常见 sink：

1. HTTP 客户端请求
2. URL 下载工具
3. 服务端转发与代理功能

常见切入点：

1. 抓取接口
2. webhook、回调、导入链接

### 2.8 表达式注入与模板注入

常见 source：

1. 模板字符串
2. 表达式片段
3. 动态规则、动态条件

常见 sink：

1. SpEL
2. OGNL
3. EL
4. 模板引擎渲染

常见切入点：

1. 后台规则配置
2. 模板自定义功能
3. 邮件、报表、通知渲染

### 2.9 CSRF 与 CORS 配置错误

常见 source：

1. 浏览器自动携带的认证信息
2. 跨域接口访问

常见 sink：

1. 缺失 CSRF 防护的敏感接口
2. 过宽的 CORS 策略

常见切入点：

1. 后台操作接口
2. 个人信息修改、转账、删除、导出等接口

### 2.10 开放重定向

常见 source：

1. `redirect` 参数
2. 回跳地址

常见 sink：

1. 重定向响应头
2. 框架重定向方法

常见切入点：

1. 登录后跳转
2. 单点登录、扫码登录

### 2.11 未授权与越权

常见 source：

1. 用户标识
2. 资源 ID
3. 角色参数

常见 sink：

1. 查询、修改、删除、导出操作

常见切入点：

1. 后台管理接口
2. 资源详情接口
3. 批量操作接口

### 2.12 敏感信息泄露

常见 source：

1. 异常对象
2. 配置文件
3. 调试接口

常见 sink：

1. 错误响应
2. 日志输出
3. 管理端点

常见切入点：

1. 全局异常处理
2. `/actuator`、调试页面、错误页面

## 3. 常见 source 分类

真实项目中常见 source 包括：

1. Query 参数
2. Form 表单
3. JSON 请求体
4. Header
5. Cookie
6. 文件上传内容与文件名
7. MQ 消息
8. 数据库中二次存储的用户可控内容
9. 第三方系统回调与 webhook
10. 外部配置、规则、模板和脚本内容

## 4. 常见 sink 分类

真实项目中常见 sink 包括：

1. SQL 执行
2. HTML/模板输出
3. 文件路径与文件系统操作
4. 命令执行
5. 表达式执行
6. 反序列化
7. 外部网络请求
8. XML 解析
9. 动态类加载与反射调用
10. 权限敏感资源访问

## 5. 使用建议

1. 审计时先找 source，再找 sink，中间再看校验和权限。
2. 如果一个接口看起来“很普通”，但输入最终能进入上述 sink，就需要继续深挖。
3. 每种漏洞都尽量用一条完整调用链来理解，而不是只记定义。
