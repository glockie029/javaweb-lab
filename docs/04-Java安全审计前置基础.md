# Java 安全审计前置基础

该文档用于补齐 JavaWeb 代码审计之前必须掌握的 Java 语言与运行时基础。很多 Web 漏洞表面发生在接口层，根因却位于反射、类加载、序列化、XML 解析、文件处理或表达式执行。

## 1. 学习目标

完成本部分后，应具备以下能力：

1. 能理解 Java 中对象、类、反射和类加载的基本运行机制。
2. 能识别常见危险 API、危险执行面和数据流落点。
3. 能为后续反序列化、XXE、命令执行、模板注入、动态调用等问题建立语言层基础。

## 2. 需要掌握的核心主题

### 2.1 反射与注解

学习重点：

1. `Class`、`Method`、`Field`、构造方法、访问控制绕过。
2. 注解读取、运行时注解、框架如何基于注解发现组件。
3. 反射调用如何把“字符串”转为“可执行逻辑”。

审计关注点：

1. 可控类名、方法名、字段名。
2. 基于注解自动注册或自动映射时的边界问题。
3. 反射结合路由、模板、任务调度、插件机制时的风险。

### 2.2 动态代理与 AOP

学习重点：

1. JDK 动态代理、CGLIB 代理。
2. 代理对象与原始对象的调用链差异。
3. 切面、拦截器、增强逻辑的生效位置。

审计关注点：

1. 鉴权、日志、事务是否依赖切面执行。
2. 某些调用路径是否绕过代理而导致安全逻辑失效。
3. 审计时不能只看目标方法，还要看代理层。

### 2.3 类加载与类路径

学习重点：

1. 类加载器层次、类路径、资源加载方式。
2. 配置文件、模板文件、类文件从哪里被加载。
3. `ClassLoader`、`URLClassLoader`、资源定位。

审计关注点：

1. 是否存在用户可控资源路径。
2. 是否存在外部类、脚本、模板、插件被动态加载。
3. 是否存在类路径污染、危险资源覆盖或任意加载。

### 2.4 序列化与反序列化

学习重点：

1. Java 原生序列化基础。
2. 对象流、缓存对象、远程调用对象和消息对象恢复。
3. `readObject`、`Serializable`、常见对象恢复方式。

审计关注点：

1. 是否存在 `ObjectInputStream`、缓存反序列化、消息反序列化。
2. 是否引入已知危险 gadget 链依赖。
3. 是否有白名单、签名、类型限制和替代方案。

### 2.5 文件 IO、路径与压缩包

学习重点：

1. `File`、`Path`、`Files`、相对路径与绝对路径。
2. 临时目录、上传目录、下载目录、资源目录。
3. 压缩包解压、遍历目录、文件复制和文件删除。

审计关注点：

1. 路径拼接、目录穿越、文件覆盖。
2. 压缩包条目路径未校验导致 `Zip Slip`。
3. 临时文件、缓存文件、导出文件是否可被攻击者利用。

### 2.6 XML、JSON 与数据解析

学习重点：

1. XML DOM、SAX、StAX 解析。
2. JSON 反序列化与对象绑定。
3. 数据格式解析如何影响对象构造与执行路径。

审计关注点：

1. XML 是否禁用外部实体、DTD 和危险特性。
2. JSON 反序列化是否允许危险类型、多态类型或松散映射。
3. 数据解析异常是否泄露结构信息。

### 2.7 表达式引擎与脚本执行

学习重点：

1. EL、SpEL、OGNL、模板表达式、脚本引擎。
2. 表达式求值和动态上下文对象。
3. 规则引擎、动态配置、脚本扩展的基本模型。

审计关注点：

1. 表达式内容是否来自用户输入或外部配置。
2. 求值上下文中是否暴露危险对象。
3. 表达式注入和模板注入的成因是否存在。

### 2.8 异常、上下文与线程相关对象

学习重点：

1. 异常传播、统一异常处理、堆栈信息暴露。
2. `ThreadLocal`、请求上下文、用户上下文。
3. 线程池、异步任务、定时任务。

审计关注点：

1. 异步执行是否绕过原有鉴权或审计逻辑。
2. 上下文对象是否被错误复用。
3. 异常是否把内部结构、SQL、路径、版本信息暴露给外部。

## 3. 常见危险 API 方向

需要对以下方向保持敏感：

1. 命令执行：`Runtime.exec`、`ProcessBuilder`
2. 反射与动态调用：`Class.forName`、`Method.invoke`
3. 类加载：自定义 `ClassLoader`、远程资源加载
4. 反序列化：`ObjectInputStream`
5. XML 解析：允许外部实体或 DTD 的解析器
6. 文件处理：可控路径、解压、复制、覆盖
7. 表达式执行：SpEL、OGNL、EL、脚本引擎

## 4. 推荐练习方式

1. 用极小示例分别验证反射调用、对象反序列化、XML 解析和路径拼接行为。
2. 对每个示例都记录：输入点、关键 API、执行结果、风险条件。
3. 不仅看“如何写”，更要明确“为什么在审计中危险”。

## 5. 达标标准

达到以下标准时，可认为本阶段完成：

1. 看到反射、动态加载、对象恢复、表达式求值等代码时能立刻识别风险方向。
2. 能解释某段代码为什么可能导致反序列化、XXE、命令执行或路径穿越。
3. 能把 Java 语言层风险和 Web 层业务入口串联起来。
