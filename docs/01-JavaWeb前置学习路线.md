# JavaWeb 前置学习路线

## 1. 路线定位

本路线服务于 Java 代码审计学习，目标不是覆盖完整开发技能，而是建立足够稳定的“工程识别能力 + 请求链理解能力 + 风险点定位能力 + 审计输出能力”。

完成本路线后，应达到以下要求：

1. 能从工程文件、依赖配置和启动方式快速判断项目技术栈。
2. 能看懂请求如何从 Web 入口进入业务层、数据层和输出层。
3. 能对主流 JavaWeb 项目进行基础到中等复杂度的代码审计。
4. 能建立 source、sink、权限链、配置链和依赖链之间的对应关系。
5. 能形成完整的问题记录、利用条件分析和修复建议输出。

## 3. 分阶段学习计划

### 阶段一：Java 安全审计前置基础

学习重点：

1. 反射、注解、动态代理、类加载、类路径和对象生命周期。
2. Java 序列化/反序列化、对象流、常见反序列化入口。
3. IO/NIO、文件路径处理、临时目录、压缩包处理。
4. XML 解析、表达式引擎、脚本执行和常见危险 API。
5. 异常传播、线程本地变量、上下文对象和配置读取。

审计视角：

1. 理解漏洞为什么会在 Java 语言层和运行时层出现。
2. 建立对反射调用、动态加载、对象恢复和表达式求值的敏感度。
3. 为后续反序列化、XXE、模板注入、命令执行、类加载类问题打基础。

建议输出物：

1. 一份 Java 审计基础笔记。
2. 一份危险 API 与触发条件清单。

### 阶段二：Java 工程化实践与构建链

学习重点：

1. `Maven` 坐标、生命周期、依赖范围、传递依赖、依赖冲突解决。
2. `pom.xml` 中的 `dependencies`、`dependencyManagement`、`plugins`、`profiles`、`properties`。
3. 多模块工程的聚合、继承、公共模块、启动模块和测试模块。
4. `Gradle` 的插件、依赖、任务、子模块和构建脚本结构。
5. `jar`、`war`、可执行包、外部容器部署和资源文件打包方式。
6. 配置文件的加载顺序、环境隔离、外部配置和敏感配置注入方式。

审计视角：

1. 从构建文件识别技术栈和高风险依赖。
2. 从多模块结构定位入口模块、公共安全模块和共享组件。
3. 从 profile 与打包方式判断真实运行形态和暴露面。

建议输出物：

1. 一份 Maven 审计要点笔记。
2. 一份 Gradle 审计要点笔记。
3. 一份“从工程文件识别技术栈”的方法清单。

### 阶段三：HTTP 与 Web 基础

学习重点：

1. `HTTP/HTTPS`、请求报文、响应报文、状态码、缓存相关头。
2. `GET`、`POST`、`PUT`、`DELETE`、`Content-Type`、`Cookie`、`Location`、`Authorization`。
3. URL 编码、表单、JSON、文件上传、重定向与转发。
4. Session、Cookie、Token、同源、跨域、浏览器自动行为。

审计视角：

1. 明确用户可控输入来自哪些位置。
2. 理解会话标识、认证凭据和浏览器行为如何影响漏洞触发。
3. 为 CSRF、CORS、认证绕过、开放重定向等问题打基础。

建议输出物：

1. 一份 HTTP 基础笔记。
2. 一份“请求参数入口分类”笔记。

### 阶段四：Servlet 容器与 JavaWeb 核心组件

学习重点：

1. `Tomcat` 基本结构、Web 应用部署模型、请求进入容器后的流转。
2. `Servlet` 生命周期、`HttpServletRequest`、`HttpServletResponse`。
3. `Filter`、`Listener`、`JSP`、`web.xml` 与注解配置。
4. 请求转发、会话管理、字符编码和异常处理。

审计视角：

1. 定位传统 JavaWeb 项目的入口点和全局处理链。
2. 区分容器层逻辑、鉴权层逻辑和业务处理逻辑。
3. 明确老项目中哪些组件最容易藏敏感处理和历史漏洞。

建议输出物：

1. 一份 Servlet 请求流转图。
2. 一份 Filter/Listener 审计关注点笔记。

### 阶段五：数据交互、输入输出与持久层

学习重点：

1. 请求参数读取、字符编码、多部分表单和文件上传处理。
2. `JDBC`、连接池、事务、预编译语句和动态 SQL。
3. 服务端模板输出、响应渲染、文件下载、对象序列化输出。
4. `MyBatis`、`JPA/Hibernate` 的基础认知和常见风险边界。

审计视角：

1. 识别 SQL 注入、文件上传、路径穿越、任意文件操作。
2. 识别输出点、模板渲染点和 XSS 风险。
3. 识别 ORM 层、动态 SQL、查询拼接和对象映射中的危险路径。

建议输出物：

1. 一份高风险输入输出链路示例。
2. 一份持久层风险笔记。

### 阶段六：认证、授权与会话安全

学习重点：

1. 登录流程、Session 鉴权、退出登录、会话失效、会话固定与会话劫持边界。
2. 传统 JavaWeb 中基于 `Servlet`、`Filter`、`Session` 的鉴权逻辑，以及“是否登录”和“是否有权访问资源”的边界。
3. RBAC、资源归属、菜单权限、接口权限、数据权限。
4. 密码存储、验证码、找回密码、二次验证等通用认证模块。

说明：

1. 本阶段优先讲传统 JavaWeb 与通用 Web 安全逻辑，不提前混入 `Spring Security`、`Shiro`、`Interceptor` 等 Spring 生态实现细节。
2. `Token` 鉴权在本阶段只作为通用对照模型理解；涉及 `Spring MVC`、前后端分离或安全框架的具体实现，统一放到阶段七展开。

审计视角：

1. 识别未授权、越权、会话固定、会话劫持和权限判断缺失。
2. 明确“是否登录”和“是否有权访问资源”不是同一个问题。
3. 建立对接口保护链、资源归属校验和权限下沉问题的判断能力。

建议输出物：

1. 一份登录鉴权链路笔记。
2. 一份权限漏洞排查项清单。

### 阶段七：Spring 生态与主流框架审计

学习重点：

1. `Spring Boot` 启动流程、自动配置、外部配置和组件扫描。
2. `Spring MVC` 请求映射、参数绑定、消息转换器、异常处理。
3. `Interceptor`、`AOP`、`ControllerAdvice`、`Validation`。
4. `Jackson`、`Fastjson`、`MyBatis`、`JPA/Hibernate`、模板引擎、`Spring Security`、`Shiro`。
5. `Actuator`、调试接口、健康检查、任务调度、异步执行、`WebSocket/SSE`。

说明：

1. 本阶段是在阶段六已经掌握通用认证、授权、会话安全逻辑的前提下，再看这些逻辑如何在 Spring 生态中落地。
2. 涉及 `Interceptor`、`Spring Security`、`Shiro`、前后端分离 `Token` 鉴权实现等 Spring 或框架特有内容，统一在本阶段处理。

审计视角：

1. 能从 `Controller` 跟踪到 `Service`、`DAO`、模板层和外部调用。
2. 能识别框架自动完成的工作与业务代码显式完成的工作。
3. 能快速定位安全组件、消息转换器、参数绑定器和敏感配置入口。

建议输出物：

1. 一份 Spring MVC 调用链说明。
2. 一份主流框架组件与典型风险映射表。

### 阶段八：漏洞类型与 Source-Sink 映射

学习重点：

1. 建立 `source -> validation/sanitization -> sink` 的通用分析框架。
2. 逐项覆盖 SQL 注入、XSS、文件上传、路径遍历、命令执行、反序列化、XXE、SSRF、SSTI、表达式注入、CSRF、开放重定向、CORS 配置错误、敏感信息泄露等问题。
3. 区分语言层风险、框架层风险、业务层风险和配置层风险。

审计视角：

1. 不只记漏洞名，而是明确进入点、触发条件、危险调用和防护缺失。
2. 能把问题归类到具体组件、具体对象或具体方法。

建议输出物：

1. 一份漏洞映射总表。
2. 每类漏洞至少一条典型调用链示例。

### 阶段九：配置、中间件、审计方法与复盘

学习重点：

1. 容器配置、反向代理、数据库、Redis、消息队列、日志组件、对象存储、文件目录。
2. 配置文件、运行参数、环境变量、敏感信息、测试配置和开发残留。
3. 固定审计流程、搜索策略、问题记录格式和案例复盘方式。

审计视角：

1. 从“工程识别 -> 入口定位 -> 参数跟踪 -> 危险调用 -> 权限校验 -> 配置复核 -> 输出结论”形成完整工作流。
2. 让每次审计都可复用，而不是只对单个项目有效。

建议输出物：

1. 一份配置与中间件审计笔记。
2. 一份标准化审计流程模板。
3. 一份案例复盘模板。

## 4. 推荐学习顺序

推荐按以下顺序推进：

1. 先完成阶段一和阶段二，解决“能不能进项目”的问题。
2. 再完成阶段三和阶段四，解决“能不能看懂请求链”的问题。
3. 然后完成阶段五和阶段六，解决“能不能看懂数据链和权限链”的问题。
4. 再进入阶段七，解决“能不能看懂主流现代项目”的问题。
5. 最后用阶段八和阶段九把漏洞映射、方法论和复盘能力固化下来。

## 5. 真实项目训练分级

在阶段一到阶段九之外，补充一条从“自写样例”过渡到“真实项目审计”的训练线。该训练线用于解决“概念已经掌握，但进入真实项目仍然无从下手”的问题。

建议按以下级别顺序推进：

1. `L1`：自写 `Servlet + JDBC` 小项目  
   目标：建立传统 JavaWeb 请求链、JDBC 执行链和最小 source-sink 习惯。  
   重点：登录与 Session、SQL 拼接、文件上传下载、异常输出、水平越权。  
   输出物：请求链图、SQL 执行链图、SQL 注入记录、水平越权记录、路径穿越记录。

2. `L2`：自写 `Spring Boot + MyBatis` 项目  
   目标：建立现代主流 JavaWeb 调用链。  
   重点：`Controller -> Service -> Mapper`、参数绑定、拦截器、动态 SQL、资源归属校验。  
   输出物：Spring MVC 请求链图、MyBatis 风险映射表、登录鉴权链路、动态 SQL 审计记录、越权复盘。

3. `L3`：小型开源 CMS / 后台系统  
   目标：从自写样例进入真实开源项目。  
   重点：技术栈识别、路由枚举、权限链、高风险接口和完整问题记录。  
   输出物：技术栈识别报告、路由清单、权限链说明、高风险接口清单、问题记录和项目复盘。

4. `L4`：中型多模块 Spring Boot 项目  
   目标：训练多模块结构、公共安全模块和权限链分析能力。  
   重点：父工程、启动模块、公共模块、安全模块、开放 API、定时任务、文件服务、数据访问层。  
   输出物：模块关系图、安全模块笔记、公共组件风险清单、权限链审计报告、数据权限问题记录、多模块复盘。

5. `L5`：历史 CVE / CNVD 项目复盘  
   目标：通过漏洞版本和修复版本对比，学习真实漏洞成因、补丁思路和变体搜索。  
   重点：补丁 diff、source-sink 调用链、修复边界、同类变体搜索。  
   输出物：漏洞成因分析、补丁 diff 分析、调用链、复现条件、修复方式、变体搜索记录、完整复盘报告。

## 6. 依赖漏洞、补丁 diff 与工具化补充

从进入真实项目训练开始，就应并行补充以下三条长期训练线；在阶段七到阶段九阶段继续加深，而不是等后期才首次接触：

1. 依赖漏洞识别  
   目标：建立第三方组件、传递依赖、漏洞版本、危险功能启用状态和可达性分析能力。  
   重点：`pom.xml` / `build.gradle`、依赖树、直接依赖与传递依赖、受影响版本、是否打入最终产物、危险 API 是否可达。  
   输出物：依赖漏洞记录、组件版本判断、可达性分析和风险分级结论。

2. 补丁 diff 复盘  
   目标：通过真实漏洞修复记录学习漏洞成因、修复方式和同类变体搜索方法。  
   重点：漏洞版本与修复版本对比、修改文件定位、修改前后调用链、修复边界和变体搜索关键词。  
   输出物：补丁 diff 复盘报告、修复边界分析、同类变体搜索记录。

3. SAST / 半自动审计工具使用  
   目标：建立规则扫描、数据流辅助、依赖扫描和批量排查能力，但坚持人工复核。  
   重点：`Semgrep`、`CodeQL`、`SpotBugs / FindSecBugs`、`OWASP Dependency-Check`、依赖树分析、密钥扫描、IDE 半自动审计。  
   输出物：工具化扫描记录、误报记录、人工确认结果和后续重点审计方向。

## 7. 学习策略

建议采用“小知识点 + 小样例 + 小审计结论”的方式推进。

每个阶段建议遵循以下节奏：

1. 先补概念，明确组件职责、运行时位置和边界。
2. 再写最小样例，验证请求、会话、数据库、模板或配置是否按预期生效。
3. 最后从审计视角总结：入口在哪里、参数怎么传、权限怎么判、危险点在哪里。
4. 每个阶段至少沉淀一份“调用链图”或“风险映射表”。

在进入真实项目后，建议同步遵循以下补充顺序：

1. 先按 `L1 -> L2 -> L3 -> L4 -> L5` 推进真实项目训练强度。
2. 在每个真实项目中同时做一次依赖漏洞识别。
3. 至少选一个历史漏洞项目做补丁 diff 复盘。
4. 在具备一定手工能力后，再引入 `Semgrep`、`CodeQL`、`Dependency-Check` 等工具做半自动辅助。

## 8. 当前建议的近期动作

优先顺序如下：

1. 先完成阶段五剩余内容，补齐 `JPA/Hibernate` 风险边界。
2. 阶段五收尾后进入阶段六，系统学习认证、授权、资源归属和会话安全。
3. 在阶段六推进过程中，开始准备 `L2` 级别的 `Spring Boot + MyBatis` 自写项目。
4. 进入真实项目训练前，先结合扩展文档准备依赖漏洞识别、补丁 diff 和工具化辅助的基础模板。

## 9. 完成标准

当以下内容全部满足时，可认为 JavaWeb 前置学习达到可开展系统审计的水平：

1. 能独立读懂 `pom.xml` 或 `build.gradle` 并判断项目技术栈。
2. 能解释请求如何进入容器、框架和业务代码。
3. 能跟踪一个用户输入从请求到数据库、文件系统、模板输出或外部请求的路径。
4. 能列出至少一套完整的认证与资源授权检查链。
5. 能识别主流 JavaWeb 组件中常见的高风险点。
6. 能对一个简单到中等复杂度的 JavaWeb 项目完成基础代码审计并形成结构化结论。
7. 能对至少一个真实开源 JavaWeb 项目输出结构化审计结果。
8. 能对至少一个历史漏洞项目完成补丁 diff 复盘。
9. 能区分“命中漏洞依赖”“存在风险但未证实可利用”和“确认可利用”三类依赖漏洞结论。
